
# 🧠 Regras Gerais de Desenvolvimento

## 📌 Código
- Sempre manter o código limpo e legível
- Seguir boas práticas (Clean Code)
- Evitar código duplicado (DRY)
- Priorizar clareza ao invés de complexidade
- Nomear variáveis, funções e classes de forma descritiva

## 🧩 Arquitetura
- Sempre modularizar o código
- Separar responsabilidades (SRP - Single Responsibility Principle)
- Evitar arquivos muito grandes
- Criar funções reutilizáveis

## 📁 Organização de Diretórios
- Manter estrutura de pastas organizada e padronizada
- Separar:
  - lógica
  - interface
  - configurações
  - utilitários
- Nomear pastas e arquivos de forma clara

## 🔄 Contexto do Projeto
- Sempre verificar se existe um arquivo `context.md`
- Se não existir:
  - Criar automaticamente
- Se existir:
  - Atualizar constantemente com:
    - Alterações feitas
    - Bibliotecas utilizadas
    - Decisões técnicas
    - Preferências do desenvolvedor

## 📚 Documentação
- Documentar partes importantes do código
- Explicar decisões não óbvias
- Manter README atualizado

## ⚙️ Dependências
- Registrar todas as bibliotecas utilizadas
- Evitar dependências desnecessárias
- Manter versões organizadas (requirements.txt ou equivalente)

## 🚀 Evolução do Projeto
- Registrar mudanças relevantes
- Evitar alterações sem histórico
- Manter rastreabilidade

## 🧠 Preferências do Desenvolvedor
- Código simples > código "inteligente"
- Legibilidade sempre vem primeiro
- Evitar overengineering
